# Отладка и просмотр данных

Здесь лежит **Markdown-снимок** справочника НПА и **скрипты** для локальной диагностики (не нужны приложению в Docker).

## Где смотреть результат работы монитора

| Что | Где |
|-----|-----|
| База состояния (проекты, история, справочник `npa_docs`) | `data/state.sqlite3` (путь задаётся `STATE_DB_PATH`) |
| Реестр проектов в Excel | `data/Output.xlsx` (`EXCEL_PATH`) |
| Снимок таблицы **npa_docs** в читаемом виде | **`debug/npa_docs_catalog.md`** |

В `npa_docs` **нет** отдельных глав **ПУЭ** — только документы с видом, органом утверждения, номером и датой (поля задаются кодом монитора, каталог из `rag_norm` не изменяется).

## Источник проектов

Для актуальных проектов доступен источник **`PROJECT_SOURCE=portal`**. Он использует живой
endpoint `https://regulation.gov.ru/api/public/PublicProjects/GetFiltered` с фильтром
`creationDate>=...` и сортировкой `-creationDate`. После успешного прогона сохраняется
`portal_last_creation_date`; следующий ручной запуск перечитает только проекты с этой даты
и новее. Новые строки в Excel подсвечиваются цветом текущего запуска.

## Сопоставление с проектами

При каждом `run_once` строится **`Matcher`**:

- Из заголовка проекта извлекаются структурные ссылки на НПА.
- Для сравнения используются только **вид документа**, **номер**, **дата** и **утвердивший орган**.
- Название документа не участвует в fuzzy-сравнении.

Итог матчинга сохраняется в `matched_docs` (JSON массив подписей, обычно `doc_display_name`) и дублируется на лист **`matches`** в Excel (строки на каждое сопоставление). После прогона метрики пишутся в **`RUN_METRICS_JSON_PATH`**.

Пересобрать справочник НПА из `All_marked_docs` и обновить SQLite:

```bash
python debug/scripts/refresh_npa_catalog.py
```

Обновить Markdown после изменения БД:

```bash
python debug/scripts/export_npa_docs_md.py
```

Статистика заполненности `npa_docs`:

```bash
python debug/scripts/npa_fill_stats.py
```

Проекты с пустыми полями в `npa_docs` (в консоль / в файл):

```bash
python debug/scripts/list_npa_gaps.py
python debug/scripts/dump_npa_gaps.py
```
